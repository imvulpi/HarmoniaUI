Detailed Description

Defines the contract for visual engines in the HarmoniaUI framework, responsible for rendering UINode instances according to their computed styles and visual resources.

Visual engines handle all drawing logic for HarmoniaUI nodes, integrating with Godotâ€™s _Draw method to produce the final on-screen visuals.

Implementations of IVisualEngine are typically assigned to a node through its StyleResource.VisualResource. If no custom engine is provided, a default implementation such as BaseVisualEngine is used.

Developers can implement custom visual engines to support specialized rendering techniques such as vector drawing, procedural textures, custom effects, or unique theming systems.
