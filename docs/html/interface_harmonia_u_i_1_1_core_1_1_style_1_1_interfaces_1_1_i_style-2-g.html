Detailed Description

Defines the contract for style objects in HarmoniaUI, used by Raw.StyleResource, Parsed.ParsedStyle, and Computed.ComputedStyle.

Template Parameters
T1Type used for single value properties (ex. width, height, position X and Y). The concrete type depends on whether the style is raw, parsed, or computed:
  • Raw style -> string
  • Parsed style -> typed structure or enum
  • Computed style -> fully resolved, pixel value
T2Type used for side based (4-value) properties (ex. padding, margin, border widths). The concrete type depends on the style stage (raw, parsed, computed), similar to T1 .

The IStyle<T1, T2> interface provides a unified API so that style properties can be accessed consistently regardless of whether the style is raw, parsed, or computed.

Direct interface properties do not require parsing or computation and can be read as is from any style stage.

Implementations of this interface should ensure that the chosen types for T1 and T2 match the intended stage of the style.
